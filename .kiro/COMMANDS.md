# Comandos Úteis - Kiro

## 📋 Specs

### Criar Nova Spec
```
"Quero adicionar [descrição da feature]"
"Preciso corrigir [descrição do bug]"
```

### Continuar Spec Existente
```
"Continue a spec [nome-da-spec]"
"Abra a spec finance-export"
```

### Executar Tarefas
```
"Execute todas as tarefas"
"Execute a tarefa 2.1"
"Execute as tarefas 1.1 até 1.3"
```

### Revisar Spec
```
"Revise o requirements.md"
"Atualize o design.md com [mudança]"
"Adicione uma tarefa para [algo]"
```

## 🎨 Steering Files

### Editar Padrões
```
"Abra o project-standards.md"
"Adicione no component-patterns.md que devemos usar [padrão]"
"Atualize git-workflow.md para incluir [processo]"
```

### Criar Novo Steering
```
"Crie um steering file para padrões de API routes"
"Adicione diretrizes de segurança em um novo steering"
```

### Referenciar Steering
```
"Seguindo #component-patterns, crie um componente"
"De acordo com #git-workflow, faça o commit"
```

## 💡 Dicas

### Contexto
Use `#` para referenciar arquivos/steering:
```
"Seguindo #component-patterns e olhando #file:types/finance.ts, crie..."
```

### Iteração
Seja específico sobre mudanças:
```
"No design.md, mude a abordagem de X para Y porque..."
```

### Múltiplas Ações
Você pode pedir várias coisas:
```
"Crie o componente, adicione as traduções e atualize o README"
```

---

💡 **Dica**: Seja natural! Kiro entende linguagem natural, não precisa decorar comandos.
